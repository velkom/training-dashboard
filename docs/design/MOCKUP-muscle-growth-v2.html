Key UX changes vs v1

  • Ring gauge replaces the badge — instant visual read of "how close am I to all muscles growing." Inspired by Apple Fitness activity rings.
  • Summary pills merge the legend and the status counts into one element — less visual noise, more information density.
  • Category labels simplified — "Push / Pull / Core / Legs" instead of "Upper body — Push". Shorter labels scan faster. Category total sets shown on the right for quick volume read per body zone.
  • Elevated surfaces — muscle rows live inside a subtle raised container with concentric border-radius (outer = inner + padding). Creates visual grouping without heavy dividers.
  • Status dots replace status pills on each row — the color-coded dot is enough when the legend/pills already explain the meaning. Removes visual clutter.
  • Gap hint inside expanded detail uses a soft callout box instead of plain text — more scannable, feels like an actionable suggestion.
  • Staggered entrance — category groups animate in with 60ms stagger, 350ms ease-out. Subtle but gives the component life on week change.

Design principles applied

  • Concentric border radius — outer muscle-surface radius (14px) = inner row radius (0) + padding (~14px). No mismatched radii.
  • Shadows over borders — card uses layered box-shadow instead of a single border for depth.
  • Tabular nums everywhere — all numbers use font-variant-numeric: tabular-nums to prevent layout shift.
  • Antialiased text — -webkit-font-smoothing: antialiased on body.
  • Interruptible transitions — all hover/active states use CSS transitions (not keyframes), so they can be interrupted mid-animation.
  • 40px+ hit areas — muscle rows have min 40px height via padding.
  • No animation on high-frequency actions — expand/collapse is instant (no accordion slide). Only the initial category load is animated.

Implementation notes for Cursor

  • Ring gauge: simple SVG with stroke-dasharray / stroke-dashoffset, calculated from growingCount / trainedCount.
  • Summary pills computed in a single useMemo counting statuses.
  • Category total sets = sum of all bucket.muscles[m].sets for muscles in that category.
  • Everything uses existing Tailwind classes and project color tokens.
